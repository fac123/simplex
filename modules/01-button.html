button

Purpose

The button module represents a clickable button.

Usage
  • Buttons can be with form submission or to trigger events.
  • The type should be set to submit when submitting a form.
  • In all other cases the type should be set to button.
  • The button module should not be used when the purpose is to send the user to a new URL. In these cases, use the button-link module.
Example
Markup
<!-- Varieties -->
<button type="button" class="button-primary">primary</button>
<button type="button" class="button-secondary">secondary</button>
<button type="button" class="button-success">success</button>
<button type="button" class="button-warning">warning</button>
<button type="button" class="button-link">link</button>

<!-- Type -->
<button type="button" class="button-primary">button</button>
<button type="submit" class="button-primary">submit</button>
<button type="reset" class="button-primary">reset</button>

<!-- Disabled attribute -->
<button type="button" class="button-primary" disabled>disabled</button>
Accessibility
  • There must be sufficient contrast between background and foreground colors
  • The content of the button should be meaningful
  • The static, focus, hover and active states of the button should be defined
  • There must be a clearly visible focus state, and the method should be consistent across all focusable elements.
