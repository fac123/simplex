input

Purpose

The input component represents an <input> element with an explicitly associated <label> element. The component also supports additional information and/or error messages associated with the input.

Usage
  • Used within forms where single-line user input required.
  • The type can include: text, tel, email, password, search, url and other valid options.
  • The hidden attribute can be applied to the <label> element but only in specific circumstances. See Accessibility notes.
  • The placeholder attribute can be added to the <input> element. See Accessibility notes.
  • The input--required attribute should be added to the parent container for all instances were the form control is required. In these instances, the required and aria-required="true" attributes must also be applied to the <input> element.
  • When errors occur, the isInvalid attribute should be injected into the parent container.
  • The input--disabled attribute can be added to the parent container if the form control needs to be disabled.
Accessibility
  • The <label> element should only be hidden in cases where there is no other choices available, and only when the form is extremely simple and its purpose is clear to sighted users.
  • For required fields, the required and aria-required="true" attributes must also be applied to the <input> element to inform Assistive Technologies.
  • The static, focus and hover states of the <input> element should be defined
  • There must be a clearly visible focus state, and the method should be consistent across all focusable elements.
Example
Markup
<!-- input -->
<div class="input">
  <label class="input__label" for="e1">
    <span class="input__content">Input</span>
    <input class="input__control" type="text" id="e1">
  </label>
</div>

<!-- input with placeholder -->
<div class="input">
  <label class="input__label" for="e2">
    <span class="input__content">Input with placeholder</span>
    <input class="input__control" type="text" id="e2" placeholder="Add Your Name">
  </label>
</div>

<!-- input with value -->
<div class="input">
  <label class="input__label" for="e3">
    <span class="input__content">Input with value</span>
    <input class="input__control" type="text" id="e3" value="Add Your Name">
  </label>
</div>

<!-- input with hidden label -->
<div class="input">
  <label class="input__label" for="e4">
    <span class="input__content hidden">Hidden input</span>
    <input class="input__control" type="text" id="e4">
  </label>
</div>

<!-- input with hint -->
<div class="input">
  <label class="input__label" for="e5">
    <span class="input__content">Input with hint message</span>
    <input class="input__control" type="text" id="e5">
    <span class="message-hint">Hint Message</span>
  </label>
</div>

<!-- input required -->
<div class="input input--required">
  <label class="input__label" for="e6">
    <span class="input__content">Input required</span>
    <input class="input__control" type="text" id="e6" required>
  </label>
</div>

<!-- input invalid -->
<div class="input isInvalid">
  <label class="input__label" for="e7">
    <span class="input__content">Input invalid</span>
    <input class="input__control" type="text" id="e7" required>
    <span class="message-error">Error Message</span>
  </label>
</div>

<!-- input disabled -->
<div class="input input--disabled">
  <label class="input__label" for="e8">
    <span class="input__content">Input disabled</span>
    <input class="input__control" type="text" id="e8" disabled>
  </label>
</div>
